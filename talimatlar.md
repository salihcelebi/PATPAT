AMAÇ
- CODEX ÖZEL TALİMATLARINI KISA TUT.
- TAKILDIĞINDA VEYA KARARSIZ KALDIĞINDA BU DOSYAYI “TEK KAYNAK” OLARAK KULLAN.
- PUTER.JS ENTEGRASYONUNU 3 MODDA (TEMPLATE/MANUAL/AUTOPILOT) TEK STANDARDLA YÖNET.

KURAL: IF / ELSE / THEN (REFERANS ZORUNLULUĞU)
- IF CODEX “NE YAPACAĞINI” TAM NETLEŞTİREMİYORSA
  THEN TALIMATLAR.md’DEKİ “SABİT SÖZLÜK + JSON ŞEMA + AUTOPILOT GATE + FALLBACK” BÖLÜMLERİNE UY.
- IF CODEX YENİ BİR KURAL EKLEMEK İSTİYORSA
  THEN ÖNCE BU DOSYADAKİ KURALLARLA ÇELİŞİP ÇELİŞMEDİĞİNİ KONTROL ET; ÇELİŞİRSE BU DOSYAYI ÖNCELİKLE UYGULA.
- IF CODEX “TÜM KODU YENİDEN YAZMA” EĞİLİMİNE GİRERSE
  THEN SADECE GEREKLİ BLOKLARI PATCH/DIFF ŞEKLİNDE ÜRET VE DİĞER SATIRLARA DOKUNMA.
- IF SPJS (PUTER.JS) KAPSAMI AŞACAK GİBİ OLURSA
  THEN SPJS’Yİ SADECE JSON KARAR/TASLAK ÜRETİMİYLE SINIRLA; DOM OKUMA VE GÖNDERME YASAK.
- IF AUTOPILOT RİSKLİ DURUMDA DEVREYE GİRMEK İSTERSE
  THEN AUTOPILOT’U BLOKE ET; ADMIN’E DEVRET; KIRMIZI UYARI GÖSTER.

0) SABİT SÖZLÜK (DEĞİŞTİRME)
- SCOD = SORUMLU CODEX: BU MADDEYİ CODEX KODLAYACAK VE KODDA 1 SATIR YORUM İLE “UYGULANDI” DİYE İSPATLAYACAK.
- SPJS = SORUMLU PUTER.JS: SADECE JSON KARAR/TASLAK ÜRETİR; DOM OKUMAZ; MESAJ GÖNDERMEZ.
- SADM = SORUMLU ADMIN: TEMPLATE/MANUAL’DA GÖNDERMEYİ ONAYLAR; MANUAL’DA TASLAĞI DÜZENLER; AUTOPILOT AÇ/KAPATIR.

ZORUNLU KULLANIM
- HER TALİMAT SATIRININ SONUNDA EN AZ 1 ETİKET: SCOD / SPJS / SADM.
- SCOD GEÇEN HER MADDE İÇİN KODDA 1 SATIRLIK YORUM ZORUNLU:
  ÖRNEK: // SCOD: AI MODU UI EKLENDİ
- SPJS GEÇEN HER MADDEDE:
  INPUT: SADECE KISA JSON (SON 8 MESAJ + KISA ORDER + MIN 4 TEMPLATE)
  OUTPUT: SADECE JSON (ŞEMA DIŞI ÇIKTI GEÇERSİZ)
- SADM GEÇEN HER MADDEDE OTOMATİK GÖNDERME YASAK.
  İSTİSNA: SADECE MODE=AUTOPILOT VE send_allowed=true VE escalate=false İSE OTOMATİK GÖNDERMEYE İZİN VER.

1) ÇIKIŞ KURALI (TOKEN/PERFORMANS)
- SADECE DIFF/PATCH ÜRET.
- SADECE GEREKLİ DOSYALARI DEĞİŞTİR.
- MAKS 8 DOSYA, MAKS 12 DEĞİŞİKLİK BLOĞU.
- AÇIKLAMA YAZMA; SADECE KOD/DIFF ÜRET.
- YORUMLARI SADECE SCOD KANITI İÇİN KISA TUT.

2) PUTER.JS ROLÜ (SPJS) — NEREDE DEVREYE GİRER?
- PUTER.JS SADECE “KARAR+TASLAK” ÜRETİR.
- MESAJLARI DOM’DAN OKUMA: SCOD (mesaj_gonderici.js).
- ŞABLON PAKETİ ÜRETME: SCOD (eslestirme_ve_sablonlar.js).
- CEVABI GÖNDERME: SCOD (mesaj_gonderici.js).
- ADMIN ONAYI: SADM (TEMPLATE/MANUAL’DA).

3) TEK JSON ŞEMASI (SPJS OUTPUT) — DEĞİŞTİRME
- intent: INFO|STATUS|REFUND|PROBLEM|OTHER
- tone: CALM|APOLOGETIC|FIRM
- send_allowed: boolean
- escalate: boolean
- is_customer_right: YES|NO|UNKNOWN
- template_id: TPL_STATUS|TPL_DONE|TPL_REFUND|TPL_PROBLEM
- reply: STRING (<=700 karakter)
- admin_note: STRING (<=300 karakter)
- ask: STRING (TEK SORU; YOKSA "")

4) PUTER.JS INPUT (CTX) — MIN TOKEN
- messages: SADECE SON 8 MESAJ {role,text,ts?}
- order: {orderId,status,dateTR,link,start,amount,remaining,serviceShort}
- templates: MIN 4 TEMPLATE {id,text}
- KİŞİSEL VERİ (TEL/MAIL/ADRES) MASKELE; GÖNDERME.

5) MINIMUM PROMPT (SPJS) — SABİT STRING
PROMPT:
"SUPPORT. READ CTX. PICK template_id. WRITE TURKISH SHORT HUMAN. SAY 'KAYITLARA GORE'. MAX 1 QUESTION. RULE: IF status IN [IADE,SORUNLU] => send_allowed=false, escalate=true. OUTPUT ONLY JSON: {intent,tone,send_allowed,escalate,is_customer_right,template_id,reply,admin_note,ask}."
- max_tokens: 400
- temperature: 0.3
- model: EN UCUZ UYGUN MODEL

6) 3 MOD DAVRANIŞI (UI)
- TEMPLATE: AI TASLAK ÜRETİR; TEXTAREA READONLY; ADMIN GÖNDERİR. SADM
- MANUAL: AI TASLAK ÜRETİR; TEXTAREA EDITABLE; ADMIN DÜZENLER+GÖNDERİR. SADM
- AUTOPILOT: AI KARAR VERİR; GATE UYGUNSA OTOMATİK GÖNDERİR. SCOD

7) AUTOPILOT GATE (KIRMIZI ÇİZGİLER)
- IF status IN [IADE,SORUNLU] THEN send_allowed=false VE escalate=true ZORUNLU.
- IF is_customer_right == UNKNOWN THEN AUTOPILOT GÖNDERMEYİ DURDUR; ADMIN’E DEVRET.
- IF escalate == true THEN OTOMATİK GÖNDERME YOK; KIRMIZI “YÖNETİCİ UYARILARI” GÖSTER.
- SADECE (mode==AUTOPILOT && send_allowed==true && escalate==false) İSE sendMessage() ÇAĞIR.

8) HATA YÖNETİMİ + FALLBACK
- IF THREAD_NOT_FOUND THEN AI ÇAĞIRMA; UI UYARI; LOG WARN.
- IF MESAJ BOŞ THEN AI ÇAĞIRMA; “MESAJ YOK” GÖSTER; LOG INFO.
- IF AI JSON PARSE FAIL THEN fallback reply=TPL_STATUS; escalate=true; admin_note="AI PARSE HATASI".
- IF SEND FAIL THEN ERROR LOG + UI UYARI; TASLAĞI KORU.

9) LOG STANDARTLARI

- PROMPT/CEVAP TAM METNİNİ LOG’LAMA; SADECE orderId, username, mode, template_id, escalate, send_allowed, reply_len LOG’LA.
